## ⚠️ 重要開發原則（本次教訓）

**一次做好不要分段。事前規劃完整後動作就一次到位。**

本次因為分段改動，導致：
1. 第一波改完部署後，發現 Tab 列手機版沒隱藏（CSS 放在 media query 內）
2. 前端月費改了但 Tab 列問題漏掉
3. Sitemap 修正需要部署但怕帶上未測試的優化
4. 多次來回部署增加風險

**正確做法：** 先完整讀完所有程式碼 → 列出完整清單 → 一次性全部改完 → 語法檢查 → 統一輸出 → 一次部署

## 一、Bug 修復（6 項）

### B1. 重複根路由衝突 ✅
- **位置：** main.py 第 520 行 vs 原第 4353 行
- **問題：** `@app.get("/")` 定義兩次，後者覆蓋前者，首頁回傳 index.html 而非 homepage.html
- **修法：** 刪除第 4353 行的重複路由

### B2. 防守位（stop_loss）計算邏輯修正 ✅
- **位置：** main.py `_do_analyze()` 防守位區段
- **問題：** `min(min_stop, max(raw_stop, max_stop))` 導致結果永遠 ≤ min_stop（-2%）
- **修法：** 改為 `max(max_stop, min(raw_stop, min_stop))` — 確保在 -2% 到 -10% 之間

### B3. K 棒型態辨識統一 ✅
- **位置：** `detect_kline_patterns()` 第 1409 行 + `detect_kbar_pattern()` 第 4540 行
- **問題：** 兩套獨立判斷，門檻不同可能產生矛盾
- **修法：** 
  - `detect_kbar_pattern()` 新增第 4 個回傳值 `win_rate`（內建 `_WIN_RATE_MAP`）
  - `_do_analyze()` 不再呼叫 `detect_kline_patterns()`，統一從 `detect_kbar_pattern` 取結果
  - `detect_kline_patterns()` 函式定義保留但不再被呼叫

### B4. 管理員密碼改環境變數 ✅
- **位置：** main.py `_db_init()` 管理員帳號區段
- **修法：** `ADMIN_PWD = os.environ.get("ADMIN_DEFAULT_PWD", "630428")`
- **部署注意：** Zeabur 環境變數需加 `ADMIN_DEFAULT_PWD`（設強密碼）

### B5. DB 連線加 context manager ✅
- **位置：** main.py `_db_conn()` 下方
- **修法：** 新增 `_db()` context manager（with 語法自動 close/commit/rollback）
- **備註：** 現有 50+ 處 `_db_conn()` 暫不動，等 A1 拆模組時逐步替換

### B6. 前端月費 $399 → $499 ✅
- **位置：** index.html 兩處按鈕文字
- **修法：** `sed -i 's/\$399／月/\$499／月/g'`

### T2. 支撐候選權重動態化 ✅

### T3. 壓力計算加量能確認 ✅

### T4. 趨勢加入 MA20 斜率判斷 ✅

### T6. 風險報酬比加勝率加權 ✅

### T7. 出場警示加「MA5 跌破 MA20」✅

- **邏輯：** 今天 MA5 < MA20 且昨天 MA5 ≥ MA20（剛死叉）→ 中級出場警示

### T9. 結論邏輯保留原架構 + 基本面追加 ✅
