## ❌ FB Page Token 今日未完成

### 問題根本原因
兩個粉專都是透過 **Business Manager** 管理，不是個人帳號直接擁有。
Graph Explorer 的 User Token `me/accounts` 一直回傳 `data: []`。

### 今日嘗試過的路（全部失敗）
1. Graph Explorer `me/accounts` → `data: []`
2. SoftGlow AutoPost（`2164339914410625`）→ `Invalid Scopes: manage_pages`
3. SoftGlow AutoPost 2（`1661900988880526`）→ `me/accounts` 仍空
4. Business Manager「連結資產」→ 只有廣告帳號選項
5. System User 建立 → Meta 名稱規則一直擋

### 正確解法（下次繼續）

**方法一：Long-lived Token 換 Page Token**（最快，不用寫程式）

步驟：
1. Graph Explorer 拿 User Token（SoftGlow AutoPost 2，`pages_show_list` + `pages_manage_posts`）
2. 用 App Secret 換 Long-lived User Token：
```
GET https://graph.facebook.com/oauth/access_token
  ?grant_type=fb_exchange_token
  &client_id=1661900988880526
  &client_secret={APP_SECRET}
  &fb_exchange_token={SHORT_LIVED_USER_TOKEN}
```
3. 用 Long-lived Token 打 `/{user-id}/accounts` → 這次才會回傳粉專列表
4. 拿到兩個粉專的 Page Token 存進 Zeabur 環境變數

App Secret 在：`developers.facebook.com/apps/1661900988880526/settings/basic/` → 顯示 App Secret

**方法二：後端 FB OAuth 端點**（備用，較複雜）

在後端加 `/auth/facebook/start` 和 `/auth/facebook/callback`，
跟 Threads OAuth 一樣的 flow，callback URL 加進 Meta App 設定。
