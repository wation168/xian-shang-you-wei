## 付款流程（全自動）

1. 用戶點「立即訂閱」→ 填 Email + 自設密碼
2. 後端 `/create_order` 產生綠界訂單，密碼暫存在後端（不傳給綠界）

## 選股生成器

### 架構（v7.4：低 FinMind 用量版，16:30）
```
每日 16:30（台北時間，交易日）
Step 1  鉅亨 RSS 近7天新聞（5個 feed，≤300則）→ 萃取股票代號（0 FinMind 呼叫）
Step 2  TWSE STOCK_DAY_ALL 成交量前100支（1次 HTTP，0 FinMind 呼叫）
Step 3  交集優先合併 → 候選清單 ≤50 支
Step 4  FinMind 技術分析（MA/KD/MACD）→ ≤75 次 FinMind 呼叫
        → 規則式評分（tech_scr 0~7 → score 50~88）
        → 輸出 stock_picker/output/latest.html
付費或 referral_unlocked=1 會員透過「選股」Tab 查看
```

## Claude Code 操作原則

1. **每次啟動先讀總結**  
   第一句話說「讀取 總結.md，這是專案說明」，讓 Claude Code 取得完整上下文後再開始工作。

2. **修改前先全面檢查**  
   告知要改的功能後，Claude Code 應先讀取所有相關檔案（backend/main.py、frontend/index.html 等），找出影響範圍再動手。

3. **修改後自我驗證**  
   每次改完，Claude Code 應自行確認：語法正確、邏輯一致、沒有殘留舊程式碼、沒有引入新 bug。

4. **大改動前確認可還原**  
   若要大幅重寫某個功能，先確認 git 有 commit 或檔案有備份，避免無法復原。  
   快速備份指令：`git add -A && git commit -m "backup before refactor"`

5. **一次給完整指令**  
   盡量把所有需求寫在同一段訊息裡，不要分段給（例如「先做 A」→ 等結果 → 「再做 B」），這樣 Claude Code 能一次規劃、減少來回。

6. **修完後列出改動摘要**  
   每次完成修改後，Claude Code 應列出：改了哪些檔案、每個檔案改了什麼，方便確認與上傳。

### 省 Token 操作原則

- 啟動時不需要每次都讀總結.md，只在第一次或需要了解整體架構時才讀
- 修改前只讀相關檔案，例如：
  - 改後端邏輯 → 只讀 backend/main.py 相關函式
  - 改前端 UI → 只讀 frontend/index.html 相關區塊
  - 改選股邏輯 → 只讀 stock_picker/ 相關檔案
- 用 grep 或 sed 定位目標區塊，不要整檔讀取
- 修改完只驗證改動的部分，不需要全檔語法檢查
