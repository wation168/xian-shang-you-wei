## 二、語言策略變化

### 原始規畫（10 語言）

| 語言 | 路徑 | 目的 |
|------|------|------|
| 繁中 | `/zh-TW/` | 起點驗證 |
| 英文 | `/`（根目錄） | CPC 最高市場 |
| 日文 | `/ja/` | 封閉高價值 |
| 西班牙文 | `/es/` | 人口紅利 |
| 韓文 | `/ko/` | Google 市佔上升 |
| 德文 | `/de/` | 歐洲核心 |
| 法文 | `/fr/` | 高消費力 |
| 葡萄牙文 | `/pt/` | 巴西新興市場 |
| 印地文 | `/hi/` | 流量大國 |
| 越南文 | `/vi/` | 東南亞爆發 |

### 現行調整（10 語言，2 個替換）

| 變化 | 原本 | 改為 | 原因 |
|------|------|------|------|
| 移除 | 印地文 `/hi/` | 印尼文 `/id/` | 印尼 Google 使用率更高，印地文市場 CPC 極低 |
| 移除 | 越南文 `/vi/` | 保留但降優先級 | 越南文內容品質 AI 生成較不穩定 |
| 新增 | — | 簡中 `/zh-CN/` | 海外華人搜尋量大，與繁中共用計算邏輯 |

**現行 10 語言：** en, zh-TW, zh-CN, ja, fr, de, es, pt, ko, id

## 三、URL 架構演進

### 原始規畫
```
/zh-TW/{slug}/          ← 目錄結構
/en/{slug}/
/ja/{slug}/
```

### 中期（generate_tools v1）
```
/tools/zh-TW/{slug}/index.html    ← 巢狀目錄
/tools/en/{slug}/index.html
```

### 現行（generate_tools v2 — 已鎖定）
```
/tools/{slug}.html                ← zh-TW 扁平，直接在 /tools/ 下
/tools/en/{slug}.html             ← 其他語言在子目錄
/tools/ja/{slug}.html
```

**為什麼改？**
- 扁平結構 URL 更短，Google 爬取效率更高
- zh-TW 作為預設語言放在根目錄，SEO 權重最高
- 與 Sitemap 和 hreflang 一致，避免重複內容判定

## 四、Google AdSense 審核策略變化

### 原始規畫：一次全站送審
- 全部語言、全部頁面一起上線
- 風險：多語言翻譯頁面被 Google 判定「低價值內容」

### 現行策略：兩段式安全送審

**第一階段（進行中）：**
- Sitemap 只暴露 en + zh-TW 工具頁
- 首頁改為內容型（工具分類 + 文章 + K棒型態 + About）
- 補齊合規頁面（Privacy / Terms / Contact / About / Disclaimer / Refund）
- Blog 48 篇 + Patterns 204 頁提供「有價值內容」支撐
- **7/3 重新提交審核，等待 3-14 天**

**第二階段（審核通過後）：**
- 啟動完整 10 語言 Sitemap
- 子目錄自動繼承主網域廣告投放權限
- 每週定量新增 1,000-2,000 頁，模擬自然生長

### 第一次審核被拒原因與修正

| 被拒原因 | 修正措施 | 狀態 |
|---------|---------|------|
| 內容價值不足 | 新增 Blog 48 篇教學文章 + Patterns 204 頁 K 棒圖解 | ✅ 已完成 |
| 首頁是 SPA 無文字內容 | 新首頁改為 content-first 設計（工具分類 + 文章入口） | ✅ 已完成 |
| Blog 404 錯誤 | 修正檔案路徑（blog_full → blog） | ✅ 已完成 |
| Sitemap 頁面不完整 | 從 3,784 頁更新到 4,039 頁（含 blog + patterns + 合規頁） | ✅ 已完成 |

## 九、技術架構演進

### 原始規畫
| 項目 | 做法 |
|------|------|
| 生成引擎 | `generate.py`（v1，基礎模板） |
| Model | 未指定 |
| 部署 | 手動上傳 Zeabur |
| CI/CD | GitHub Actions 排程上傳 |

### 現行做法
| 項目 | 做法 | 變化原因 |
|------|------|---------|
| 生成引擎 | `generate_tools_v2.py`（完整 SEO 模板 + 品質檢查） | v1 文章太短、UI 太陽春 |
| Model | Claude Haiku 4.5 | 品質/價格最佳平衡 |
| JSON 修復 | 截斷回應自動修復（找最外層 `{}`） | Haiku 偶爾輸出不完整 JSON |
| CJK 計數 | 中日韓用字元數，拉丁語用 word count | `.split()` 對中文計數錯誤 |
| 快取機制 | `output/.cache/{id}_{lang}.json` | 中斷後可續跑，不浪費 API |
| HTTP 呼叫 | `requests` + monkey-patch 修復 latin-1 | Python 3.14 + Windows encoding bug |
| 部署 | 手動上傳 Zeabur（AdSense 審核期間凍結線上） | 審核期不動線上 |

## 十二、關鍵學到的教訓

| 教訓 | 說明 |
|------|------|
| CJK 字數不能用 `.split()` | 中文空白很少，`.split()` 算出 50 字但實際 1,500+ 字 |
| API 截斷要有修復機制 | `max_tokens` 要開大 + JSON 自動修復邏輯 |
| URL 結構要先鎖定 | 事後改 URL 要連動 canonical、hreflang、Schema、內部連結 |
| 快取是生命線 | 中斷重跑不重新呼叫 API，省錢省時間 |
| AdSense 審核要有耐心 | 不要在審核期間大改網站，會拉長審核 |
| 先上線再優化 | 用 Haiku 4.5 跑完全部，等賺錢了再升級熱門頁面 |
| API Key 要確認 | `sk-ant-api03-你的key` ← 不要留中文佔位符 |
